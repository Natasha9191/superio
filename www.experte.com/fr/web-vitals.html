Bulk Core Web Vitals Test : Analysez jusqu'à 500 pages

Google a présenté en 2020 les Core Web Vitals, c'est à dire trois nouvelles mesures permettant d'analyser automatiquement une partie de l'Expérience Utilisateur (User Experience, ou UX) d'un site internet. Avec le test Core Web Vitals d'EXPERTE.com, vous pouvez automatiquement analyser les Core Web Vitals d'une grande variété de pages. Notre outil vous permet d'identifier les sous-pages lentes de votre site ou de le comparer avec ceux de vos concurrents.

Description

Notre test Core Web Vital propose 3 différents modes :

  • Exploration du site internet
    Vous indiquez l'URL de départ et nous explorons le site internet pour trouver d'autres URL. L'exploration est limitée de 60 à 500 URL.

  • Liste d'URL
    Vous nous soumettez une liste d'URL, pour lesquelles nous déterminons ensuite les Core Web Vitals.

  • Mots clés
    Vous nous donnez un mot clé pour lequel nous déterminons les 50 premiers résultats sur Google.fr. Pour chaque URL, nous déterminons ensuite les Core Web Vitals.

De plus, vous avez le choix entre analyser les Core Web Vitals d'une version mobile (Mobile) ou d'un ordinateur (Ordinateur).

Core Web Vitals

Les métriques correspondent exactement à celles de Google PageSpeed Insights ou de l'outil Lighthouse Performance. Dans ce qui suit, nous allons vous présenter les différentes métriques.

1.

Largest Contentful Paint (LCP)

Le Largest Contentful Paint indique le moment où le navigateur affiche le plus grand élément visible, c'est à dire l'instant probable de chargement du contenu principal de la page.

  • Bon
    Le LCP est de moins de 2,5 secondes.

  • Optimisation requise

    Le LCP se situe entre 2,5 et 4 secondes.

  • Lent
    Le LCP est supérieur à 4 secondes.

2.

First Input Delay (FID)

Le First Input Delay indique le temps entre la première interaction de l'utilisateur et la réaction du navigateur. Un clic sur une icône est un exemple d'interaction.

  • Bon
    Le FID est inférieur à 100 ms.

  • Optimisation requise
    Le FID est compris entre 100 ms et 300 ms.

  • Lent
    Le FID est supérieur à 300 ms.

3.

Cumulative Layout Shift (CLS)

Le Cumulative Layout Shift mesure la somme de tous les décalages de mise en page se produisant pendant la durée de vie d'une page. Un décalage de mise en page se produit lorsqu'un élément visible change de position d'une image à l'autre.

  • Bon
    Le CLS est inférieur à 0,1.

  • Optimisation requise
    Le CLS est compris entre 0,1 et 0,25.

  • Lent
    Le CLS est supérieur à 0,25

Questions fréquentes

Pourquoi utiliser notre test Bulk Core Web Vital ?
⊖

Contrairement à presque tous les autres outils, notre solution teste jusqu'à 500 sous-pages à la fois. Vous pouvez donc analyser l'ensemble de votre site Web et identifier les sous-pages les plus lentes.

Les données sont-elles fiables ?
⊖

Le test Core Web Vital d'EXPERTE.com utilise les mêmes données que celles sur lesquelles se base Google PageSpeed Insights. Les résultats sont donc comparables à 100%.

Quelles sont les limites de l'outil ?
⊖

Pour éviter les abus, nous limitons actuellement la durée d'exploration à 60 ou 500 URL. Le nombre d'URL par mot-clé est limité à 50. Nous nous réservons le droit d'ajuster ces limites à tout moment.
