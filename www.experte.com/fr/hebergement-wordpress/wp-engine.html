Avis WP Engine : L'hébergeur WordPress le plus rapide du marché ?

Auteur
Janis von Bleichert
Dernière mise à jour

« A smarter way to WordPress », voilà ce que promet WP Engine. Le fournisseur de solutions d'hébergement géré pour WordPress s'occupe des aspects techniques et administratifs de votre site WordPress afin que vous puissiez vous concentrer sur la construction et l'exploitation de votre site. Mais que vaut l'offre de WP Engine ?

Nous avons examiné WP Engine de plus près et vous révélons dans cet article s'il rend les projets WordPress vraiment plus « smart ».

Qu'est-ce que WP Engine?

WP Engine est un fournisseur d'hébergement web basé à Austin, Texas, qui a été fondé en 2010 et qui gère aujourd'hui plus de 1,5 million de sites web pour plus de 185 000 clients. Comme son nom l'indique, WP Engine se concentre à 100 % sur l'hébergement WordPress, c'est pourquoi son offre est entièrement adaptée aux besoins des utilisateurs de WordPress.

WP Engine affirme être l'hébergeur WordPress le plus rapide et le plus sûr du marché. Mais le fournisseur tient-il vraiment ses promesses ? Notre test le révèle.

1.

Configuration & utilisation

Tout d'abord, il convient de noter que WP Engine est exclusivement disponible en anglais. Ni le site web ni le portail client n'ont encore été localisés en français. Si la barrière de la langue est un problème, nous vous recommandons de choisir d'autres hébergeurs qui proposent des solutions similaires. Kinsta ou HostPress pourraient par exemple vous intéresser.

Si l'anglais ne vous pose pas de problème, vous pouvez créer un compte et souscrire à un tarif. Vous pouvez également ajouter quelques add-ons supplémentaires. Vous choisissez si vous voulez payer mensuellement ou économiser avec un tarif annuel.

Le point « Data center » dans la zone « Account information » est important. Ici, vous pouvez choisir l'emplacement de votre serveur. Par défaut, les États-Unis sont choisis. Malheureusement, le fournisseur ne propose pas de serveurs en France.

Il n'y a pas de localisation française pour WP Engine, ni de serveurs français.

Une fois que vous avez créé un compte et souscrit à un tarif (pour en savoir plus sur les prix et les tarifs de WP Engine, voir ci-dessous), vous pouvez créer votre première installation WordPress.

Installation de WordPress

Avec WP Engine, l'installation d'instances de WordPress est un jeu d'enfant. Dans le portail client, cliquez d'abord sur « Add a new site » pour commencer l'installation. Il vous suffit ensuite d'entrer vos données de connexion et WP Engine se charge du reste. Votre site WordPress devrait être prêt à être lancé en quelques minutes.

L'installation de WordPress ne prend que quelques minutes.

Tableau de bord épuré, utilisation intuitive

Une fois l'installation terminée, vous trouverez votre nouveau site WordPress sous « My Sites » dans le portail client WP Engine. Il est très facile de s'y retrouver : dans le menu principal à gauche, vous naviguez vers les domaines supérieurs tels que vos sites web, les projets de commerce électronique, la gestion des utilisateurs, les add-ons complémentaires et les factures.

Dans la barre de menu supérieure, vous trouverez une fonction de recherche, ainsi que des liens vers le centre d'aide et votre profil d'utilisateur avec d'autres paramètres. Si vous avez besoin d'une aide rapide, vous pouvez contacter le chat d'assistance en bas à droite.

La zone d'administration de WP Engine est structurée de manière logique.

La liste de vos sites WordPress contient quelques informations de base sur l'installation correspondante, par exemple la version PHP ou la bande passante. Un lien direct pour se connecter à la console WordPress est également disponible. Pour ouvrir les paramètres WordPress sur WP Engine, cliquez sur le nom du projet.

Cette zone de gestion est également bien structurée. Tout d'abord, vous atterrissez dans une fenêtre intitulée « Overview » qui affiche les statistiques et les paramètres les plus importants de votre installation.

Sur la gauche, vous trouverez un menu de navigation divisé en trois environnements pris en charge pour chaque site WordPress : Production, Staging et Development (dans un premier temps, seul un environnement Production est créé). Toutes les fonctions importantes, des sauvegardes aux certificats SSL en passant par les fonctionnalités de mise en cache, sont disponibles.

Vous pouvez effectuer des réglages avancés de votre site WordPress dans ce sous-menu.

Migration de votre site WordPress via plug-in

Vous avez déjà un site WordPress et vous souhaitez le migrer vers WP Engine ? Pour cela, le fournisseur a développé son propre un plug-in appelé WP Engine Automated Migration. Vous devez l'installer sur le site WordPress que vous souhaitez migrer.

Pour effectuer la migration, vous devez installer une nouvelle instance de WordPress dans le tableau de bord de WP Engine. Ensuite, dans la console d'administration du site WordPress que vous souhaitez migrer, ouvrez le plug-in WP Engine Automated Migration et suivez les instructions. Pour cela, vous avez besoin de quelques informations que vous trouverez dans le tableau de bord WP Engine dans la vue détaillée de votre installation WordPress sous Staging > Site migration.

La migration n'est donc pas aussi « automatique » que chez certains autres hébergeurs WordPress, mais vous devez l'effectuer manuellement avec le plug-in. Néanmoins, le transfert proprement dit à l'aide de ce plug-in est très simple, même pour les utilisateurs moins expérimentés sur le plan technique.

Le plug-in WP Engine Automated Migration vous aide à migrer vos projets WordPress.

Le concept d'utilisation de WP Engine fonctionne bien. La plateforme marque des points avec des tableaux de bord propres et une navigation intuitive. Les fonctions les plus importantes de chaque instance WordPress, par exemple la connexion à la console WordPress, sont particulièrement faciles d'accès et vous ne cherchez pas longtemps les paramètres dans les sous-menus.

La déception est d'autant plus grande que l'interface utilisateur n'est pour l'instant disponible qu'en anglais, car pour certains utilisateurs francophones, la barrière de la langue pourrait être problématique.

Caractéristiques techniques & fonctionnalités

Mais alors, qu'en est-il sous la surface brillante et élégante de WP Engine ? C'est là que se décide finalement ce qui est vrai des promesses de performance et de sécurité du fournisseur. Nous avons examiné les spécifications techniques en détail avant de révélé plus loin ce qu'il en est de la sécurité et de la performance dans la pratique.

PHP seulement jusqu'à la version 8.0

En ce qui concerne la version PHP, WP Engine est malheureusement un peu à la traîne. La version 8.0 de PHP est activée par défaut, et lorsque nous essayons de changer de version, nous constatons que seul un downgrade vers la version 7.4 est possible. Chez d'autres fournisseurs, la version 8.1, plus récente, était déjà prise en charge depuis longtemps au moment de notre test, certains proposaient même la version 8.2.

Chaque nouvelle version de PHP apporte son lot de nouvelles fonctionnalités et d'améliorations, tant au niveau des performances que de la sécurité des sites WordPress. Il est donc dommage que WP Engine ne soit pas à la hauteur de ses concurrents dans ce domaine.

Vous pouvez adapter la version de PHP, mais les versions plus récentes 8.1 (au moment de notre test) et 8.2 n'étaient malheureusement pas encore prises en charge.

Serveur NGINX et Apache, bases de données MySQL

WP Engine utilise des serveurs de NGINX et Apache et des bases de données de MySQL. Malheureusement, il n'y a qu'une seule base de données par site WordPress. Vous ne pouvez pas en créer d'autres (même en payant). Sur ce point, d'autres fournisseurs d'hébergement sont nettement plus flexibles.

Les réglages avancés de la base de données ne sont pas possibles dans le tableau de bord WP Engine, mais vous pouvez accéder directement à votre base de données via phpMyAdmin.

Avec phpMyAdmin, vous pouvez accéder directement à votre base de données.

Fonctions de performance de WP Engine

WP Engine utilise exclusivement des disques durs SSD rapides, tant pour les bases de données que pour le stockage. En outre, la plateforme prend en charge le protocole HTTP/2, ce qui permet une transmission plus efficace des données sur le réseau par rapport au protocole HTTP.

La mise en cache fait appel à une technologie propriétaire appelée EverCache : elle met automatiquement en cache le contenu statique de votre site web, de sorte qu'il ne doit pas être rechargé à chaque fois qu'un visiteur se rend sur votre site. EverCache est entièrement optimisé pour WordPress, c'est pourquoi WP Engine promet une augmentation de la vitesse de page jusqu'à 40% par rapport à d'autres solutions.

Vous trouverez quelques autres fonctions de mise en cache et la possibilité de supprimer manuellement les caches dans le menu « Cache » de la zone d'administration de votre instance WordPress :

WP Engine optimise les performances de votre site avec EverCache.

Autres fonctions de WP Engine

Ce n'était bien sûr qu'un début, car WP Engine dispose de nombreuses autres fonctions. En voici quelques-unes :

  • Environnements de « staging » et de « development » : Il existe un environnement de staging où vous pouvez tester librement les mises à jour ou les modifications à l'avance. D'autres environnements de développement peuvent également être ajoutés gratuitement.

  • Accès SFTP et SSH : Vous pouvez accéder à vos données via un accès SFTP et SSH et les adapter librement. Même le wp-config,php est modifiable via SFTP.

  • Content Delivery Network (CDN) : WP Engine a intégré un CDN de Cloudflare dans tous ses tarifs. Le fournisseur promet ainsi des temps de chargement rapides pour les visiteurs du site web dans le monde entier.

  • Multi-sites : WP Engine propose les multi-sites, c'est-à-dire la gestion de plusieurs sites web dans une seule instance de WordPress. Cela n'est toutefois possible qu'à partir du tarif Professional.

En outre, WP Engine met à disposition quelques add-ons complémentaires payants qui ne sont inclus dans aucun tarif de manière standard. Le service GeoIP, par exemple, veille à ce que le contenu soit livré de manière dynamique aux visiteurs en fonction de leur localisation, ce qui améliore encore les performances. WP Engine propose par ailleurs une boîte à outils pour les freelances et les agences qui construisent des sites web pour des clients, ainsi qu'un gestionnaire de plug-ins intelligent.

À propos des plug-ins : il existe une liste assez détaillée de plug-ins interdits par WP Engine. Le mieux est de jeter un coup d'œil pour voir si l'un d'entre eux est important pour votre site.

Dans l'ensemble, WP Engine offre une bonne gamme de fonctionnalités. Il est dommage qu'aucun des tarifs ne comprenne un domaine inclus, pas plus que l'hébergement d'e-mails. La limitation à une base de données par site web est également regrettable.

Sécurité

WP Engine prétend offrir « l'hébergement WordPress le plus sûr » du marché. Il est bien sûr plus difficile d'établir des comparaisons solides avec d'autres fournisseurs dans ce domaine que dans celui des performances, car la sécurité s'exprime difficilement en pourcentages.

Sur le front de la conformité, WP Engine marque des points avec le respect des normes SOC-2 et ISO 27001-2013. Il est juste dommage que les clients français ne puissent pas choisir de serveurs en France.

Les certificats SSL gratuits de Let's Encrypt sont inclus dans tous les tarifs. Dès que vous avez attribué un domaine à votre installation WordPress, vous pouvez configurer les certificats SSL dans le sous-menu du même nom dans votre environnement de production. Il est bien sûr possible d'y importer d'autres certificats.

WP Engine met gratuitement à disposition des certificats SSL de Let's Encrypt.

Protection contre les attaques et les logiciels malveillants

WP Engine dispose d'un certain nombre de fonctions de protection intégrées pour contrer de manière proactive les attaques contre votre site. Les mises à jour de WordPress et de PHP sont automatiques dans tous les tarifs, de sorte que les versions obsolètes n'offrent pas de surface d'attaque. Il existe également une fonction de détection des menaces qui vous signale les dangers potentiels.

La visibilité est assurée par WP Engine grâce aux droits des utilisateurs et aux journaux d'accès : vous pouvez définir exactement quels utilisateurs ont accès à quelles zones et consulter les activités de votre site WordPress sous « Access Logs ».

Les journaux révèlent qui a fait quoi et quand sur votre site web.

Fonctions de sécurité avancées derrière les péages de lecture numérique

Malheureusement, WP Engine place les fonctions de sécurité les plus intéressantes derrière des péages de lecture numérique. Un pare-feu pour les applications web et une protection avancée contre les DDoS ne sont par exemple disponibles qu'en utilisant une extension payante. Cette dernière veille également à ce que vos plug-ins soient automatiquement mis à jour.

Fonctions de sauvegarde de WP Engine

Les sauvegardes sauvent la mise au cas où quelque chose irait mal sur votre site WordPress. WP Engine assure la sécurité grâce à des sauvegardes automatiques et à la possibilité de créer des mises à jour manuelles. Les sauvegardes automatiques sont effectuées quotidiennement, les sauvegardes manuelles peuvent être créées à tout moment.

Toutes vos sauvegardes – et la possibilité de restaurer votre site WordPress à partir d'une sauvegarde – se trouvent dans la gestion de votre projet WordPress sous « Backups ». Les sauvegardes sont conservées jusqu'à 60 jours, ce qui est généreux par rapport à d'autres fournisseurs.

WP Engine effectue une sauvegarde automatique quotidienne de votre site WordPress.

Dans l'ensemble, WP Engine répond aux principales normes de sécurité. Jusqu'à présent, le fournisseur n'a pas connu de fuites ou de gros scandales à cet égard. Nous apprécions le fait que les sauvegardes automatiques soient conservées jusqu'à 60 jours. Il est toutefois un peu dommage que certaines fonctions de sécurité importantes, comme la protection contre les DDoS et un pare-feu, ne soient proposées que moyennant un supplément.

Performance

Afin d'assurer le bonheur de vos visiteurs, votre site WordPress ne doit pas seulement être sûr, mais aussi rapide. C'est ce que nous vérifions dans notre test de performance. Nous comparons les performances des hébergeurs de notre test sur la base des quatre indicateurs les plus importantes : vitesse de réponse, vitesse de page, test de charge et disponibilité.

Comparabilité des fournisseurs

Pour que les résultats soient les plus comparables possibles, nous avons choisi pour chaque fournisseur un tarif qui se rapproche le plus possible d'un prix mensuel de 20 €. Chez WP Engine, le choix s'est donc porté sur le tarif Managed WordPress à 19 € par mois.

Vitesse de réponse (Time to First Byte)

Nous mesurons la vitesse de réponse du serveur à l'aide du Time to First Byte. Ce « temps jusqu'au premier octet » révèle combien de temps s'écoule entre l'envoi d'une requête par l'utilisateur (donc un visiteur de votre site web) et la livraison du premier octet de données.

Idéalement, la vitesse de réponse devrait être constante, mais dans la pratique, il y a souvent des fluctuations. Par exemple parce que d'autres sites web se trouvent également sur le serveur et influencent la performance.

Nous avons testé le Time to First Byte pendant six semaines avec le cache désactivé afin d'obtenir à la fois une valeur moyenne et une impression des variations dans le temps. Voici le résultat :

En testant la vitesse de réponse, il y a eu quelques fluctuations vers le haut, mais dans l'ensemble, le Time to First Byte était bon.

Vitesse de page (Largest Contentful Paint)

Mais un seul octet ne suffit pas à rendre un site web performant. Une meilleure mesure de la performance globale et de la vitesse de chargement proprement dite est donc le Largest Contentful Paint (LCP). Il indique combien de temps il faut à la page pour charger le plus grand élément de contenu. Comme les images, CSS et Javascript sont également pris en compte, le LCP est un bon indicateur de l'interaction de toutes les optimisations de performance.

Google est tout à fait d'accord, car le LCP fait partie des Core Web Vitals (signaux web essentiels), qui donnent des informations sur la convivialité d'un site web. D'un point de vue SEO, une bonne vitesse de page est donc également importante.

Support client

Une bonne assistance est particulièrement importante pour l'hébergement géré de WordPress, car les clients veulent éviter autant que possible les tâches administratives manuelles. Deux aspects sont importants à cet égard : un bon centre d'aide avec des guides et des tutoriels, ainsi qu'une assistance directe rapide via différents canaux.

Commençons par le centre d'aide : vous pouvez y accéder en cliquant sur l'icône d'assistance en haut à droite dans la barre de menu. Vous y trouverez une fonction de recherche qui vous permettra de trouver rapidement des articles pertinents concernant votre problème. En outre, le contenu du centre d'aide est divisé en différentes catégories, comme par exemple « Setup a Site », « WordPress Help » ou « Troubleshoot ».

Les articles eux-mêmes sont de qualité : chaque article dispose d'une table des matières, ce qui permet de mieux s'y retrouver, et des captures d'écran et des hyperliens rendent les instructions plus claires.

Le centre d'aide de WP Engine.

Le support direct est assuré par le chat en direct de WP Engine. Nous avons toujours été mis en relation rapidement avec des collaborateurs sympathiques qui semblaient bien connaître les spécifications techniques et les fonctions de WP Engine. Le service d'assistance en ligne est disponible 24 heures sur 24.

Il est dommage qu'il n'y ait pas d'autres possibilités de contact. WP Engine ne propose par exemple une hotline téléphonique que pour la vente.

Tarification

Les tarifs de WP Engine sont divisés en tarifs WordPress gérés réguliers et en tarifs e-commerce, ces derniers ne se différenciant (outre le prix) que par les fonctions e-commerce intégrées.

Quatre tarifs sont disponibles : Startup, Professional, Growth et Scale. Les différences concernent les sites web intégrés (1, 3, 10, 30), les visiteurs recommandés par mois (25 000, 75 000, 100 000, 400 000), l'espace mémoire dédié (10 Go, 15 Go, 20 Go, 50 Go) et la bande passante mensuelle (50 Go, 125 Go, 200 Go et 500 Go).

Les multi-sites ne sont pris en charge qu'à partir du tarif Professional.

Questions fréquentes & réponses

WP Engine est-il adapté aux clients français ?
⊖

WP Engine est certes adapté aux clients français, mais le fournisseur ne propose pas de serveurs en France et l'interface utilisateur n'est pour l'instant disponible qu'en anglais.

WP Engine performe-t-il bien ?
⊖

Les performances de WP Engine sont excellentes : lors du test de performance réalisé par EXPERTE.com, l'hébergeur a fourni d'excellentes vitesses de page et de réponse, même sous charge, ainsi qu'un uptime parfait, ce qui lui a valu la première place dans la comparaison avec 14 fournisseurs.

Quels sont les inconvénients de WP Engine ?
⊖

WP Engine est cher par rapport à d'autres fournisseurs. De plus, il ne couvre pas toutes les fonctions : les domaines ne sont par exemple pas intégrés, un pare-feu pour les applications web et une protection contre les DDoS sont disponibles que contre un supplément.

WP Engine propose-t-il une version d'essai gratuite ?
⊖

WP Engine ne propose pas de version d'essai gratuite. En revanche, vous pouvez demander un remboursement pendant 60 jours si vous n'êtes pas satisfait du service.
